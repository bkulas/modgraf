Wskazówki dla programistów algorytmów

1. Przygotowanie środowiska

Wymagania:

  • Java 7.x+
  • Git
  • Maven 3.x

Budowanie Modgrafa z konsoli:

  1. Pobranie repozytorium Git:
    git clone https://github.com/duboti/modgraf.git
  2. Budowanie za pomocą Maven 3:
    mvn clean install
  3. Uruchomienie programu:
    java -jar target/modgraf-4.x-jar-with-dependencies.jar

Osobiście polecam polecam budowanie Modgrafa z IDE posiadającego wtyczkę do Gita i Mavena. Wtedy po pobraniu repozytorium należy:

  1. Wybrać opcję Import Maven project
  2. Uruchamiać za pomocą Run Java Application z ustawionym parametrem main class: modgraf.Main

2. Implementacja algorytmu w trybie zwykłym

Wskazówki z przykładami dla osób chcących napisać własne algorytmy:

  1. Każdy algorytm musi dziedziczyć po klasie ModgrafAbstractAlgorithm.
    1package modgraf.algorithm; 
    2 
    3import modgraf.algorithm.ModgrafAbstractAlgorithm; 
    4 
    5public class MyAlgorithm extends ModgrafAbstractAlgorithm { ... }
  2. Należy stworzyć publiczny konstruktor przyjmujący obiekt klasy Editor.
    1public MyAlgorithm(Editor e) { 
    2    super(e); 
    3}
  3. Klasa ModgrafAbstractAlgorithm zawiera w deklaracji interfejs ActionListener, ale go nie implementuje. Faktyczna implementacja powinna znaleźć się w klasie pochodnej. W najprostszym przypadku (gdy nie istnieją żadne ograniczenia odnośnie typu grafu) implementacja metody actionPerformed(...) może ograniczyć się do wywołania metody openParamsWindow() z klasy bazowej.
    1@Override 
    2public void actionPerformed(ActionEvent arg0) { 
    3    openParamsWindow(); 
    4}
  4. Zwykle jednak istnieją ograniczenia, dlatego warto na samym początku sprawdzić typ grafu.
    1@Override 
    2public void actionPerformed(ActionEvent arg0) { 
    3    if (editor.getGraphT() instanceof DirectedGraph 
    4        && editor.getGraphT() instanceof WeightedGraph) 
    5                openParamsWindow(); 
    6    else 
    7        JOptionPane.showMessageDialog(editor.getGraphComponent(), 
    8                lang.getProperty("warning-wrong-graph-type")+ 
    9                lang.getProperty("alg-mf-graph-type"), 
    10                lang.getProperty("warning"), JOptionPane.WARNING_MESSAGE); 
    11}
  5. Jeżeli algorytm nie potrzebuje parametrów początkowych to zamiast metody openParamsWindow() używamy metody startAlgorithmWithoutParams().
    1@Override 
    2public void actionPerformed(ActionEvent arg0) { 
    3    startAlgorithmWithoutParams(); 
    4}
  6. Metoda getName() jest abstrakcyjna, więc zawsze trzeba ją zaimplementować. String zwracany przez tę metodę zostanie umieszczony w menu Algorytmy.
    1@Override 
    2public String getName() { 
    3    return lang.getProperty("menu-algorithm-maximum-flow"); 
    4}
  7. Metoda findAndShowResult() także jest abstrakcyjna, więc zawsze trzeba ją zaimplementować. W celu zwiększenia czytelności kodu warto oddzielić od siebie:
    • obliczanie wyniku (w przykładzie metoda findResult)
    • tworzenie tekstowego rozwiązania (w przykładzie metoda createTextResult(result))
    • rysowanie rozwiązania na grafie (w przykładzie metoda createGraphicalResult(result))
    1@Override 
    2protected void findAndShowResult() { 
    3    List<DefaultEdge> result = MyAlgorithm.findResult( 
    4            editor.getGraphT(), startVertex, endVertex); 
    5    if (result != null) { 
    6        createTextResult(result); 
    7        createGraphicalResult(result); 
    8    } 
    9    else 
    10        JOptionPane.showMessageDialog(editor.getGraphComponent(), 
    11            lang.getProperty("message-no-solution"), 
    12            lang.getProperty("information"), JOptionPane.INFORMATION_MESSAGE); 
    13}
  8. Jeżeli wynik działania algorytmu jest w postaci listy krawędzi to implementacja metody createGraphicalResult(result) jest bardzo prosta:
    1private void createGraphicalResult(List<ModgrafEdge> result) { 
    2    int width = 4; //Pożądana grubość krawędzi 
    3    for (ModgrafEdge edge : result) 
    4        changeEdgeStrokeWidth(edge, width); 
    5    editor.getGraphComponent().refresh(); 
    6}
  9. Analogicznie wygląda implementacja metody createGraphicalResult (result) w przypadku, gdy rozwiązaniem jest lista wierzchołków:
    1private void createGraphicalResult(List<Vertex> result) { 
    2    int width = 4; //Pożądana grubość obramowania wierzchołka 
    3    for (Vertex vertex : result) 
    4        changeVertexStrokeWidth(vertex, width); 
    5    editor.getGraphComponent().refresh(); 
    6}
  10. Metoda createGraphicalResult(result) powinna w ostatniej linijce zawierać wywołanie metody refresh().
    1editor.getGraphComponent().refresh();
  11. Równie prosto można zaimplementować metodę createTextResult (result) w przypadku, gdy chcemy po prostu wypisać listę wierzchołków:
    1private void createTextResult(List<Vertex> result) { 
    2    StringBuilder sb = new StringBuilder() 
    3            .append(lang.getProperty("menu-algorithm-my-algorithm")) 
    4            .append(":\n"); 
    5    for (Vertex vertex : result) { 
    6        sb.append(vertex.getName()); 
    7                .append(", "); 
    8    } 
    9    sb.replace(sb.length()-2, sb.length(), "."); 
    10    editor.setText(sb.toString()); 
    11}
  12. Analogicznie wygląda implementacja metody createTextResult (result) w przypadku, gdy rozwiązaniem jest lista krawędzi:
    1private void createTextResult(List<ModgrafEdge> edges, double totalWeight) { 
    2    StringBuilder sb = new StringBuilder() 
    3            .append(lang.getProperty("alg-my-message-1")) 
    4            .append(totalWeight) 
    5            .append(lang.getProperty("alg-my-message-2")); 
    6    for (ModgrafEdge edge : edges) { 
    7        sb.append(edge.getSource().getName()) 
    8                .append("-") 
    9                .append(edge.getTarget().getName()) 
    10                .append("\n"); 
    11    } 
    12    editor.setText(sb.toString()); 
    13}
  13. Metoda createTextResult(result) powinna w ostatniej linijce zawierać wywołanie metody setText(String).
    1editor.setText(sb.toString());
  14. Podczas implementacji algorytmów może być przydatne stworzenie własnych klas odpowiadających wierzchołkom lub krawędziom zawierających dodatkowe pola. W przypadku wierzchołków sprawa jest prosta, bo zawsze należy dziedziczyć po klasie Vertex. Przykład: statyczna klasa DistanceVertex znajdująca się wewnątrz klasy DijkstraAlgorithm.

    Sprawa jest dużo bardziej skomplikowana dla krawędzi, ponieważ w Modgrafie jest wiele klas reprezentujących różne rodzaje krawędzi. Ogólna zasada mówi że powinno się dziedziczyć po klasie, która została wykorzystana do tworzenia krawędzi w danym grafie:

    • graf nieskierowany nieważony — UndirectedEdge
    • graf nieskierowany ważony — WeightedEdgeImpl
    • graf nieskierowany podwójnie ważony — DoubleWeightedEdgeImpl
    • graf skierowany nieważony — DirectedEdge
    • graf skierowany ważony — DirectedWeightedEdge
    • graf skierowany podwójnie ważony — DirectedDoubleWeightedEdge

    Przykłady:

    • klasa ColoredEdge rozszerza klasę UndirectedEdge o pole kolor
    • klasa DirectedTripleWeightedEdge rozszerza klasę DirectedDoubleWeightedEdge o pola flow i consistentWithFlow
  15. Dodanie na stałe nowego algorytmu do Modgrafa będzie wymagało zmian w klasach MenuBar oraz Language w sposób analogiczny do istniejących już algorytmów.

3. Implementacja algorytmu w trybie krokowym

Wskazówki z przykładami dla osób chcących napisać własne algorytmy w trybie krokowym:

  1. Każdy algorytm w trybie krokowym musi dziedziczyć po klasie ModgrafAbstractStepsAlgorithm.
    1package modgraf.algorithm.steps; 
    2 
    3import modgraf.algorithm.ModgrafAbstractStepsAlgorithm; 
    4 
    5public class MyStepsAlgorithm extends ModgrafAbstractStepsAlgorithm { ... }
  2. Wskazówki od 2. do 6., 14. oraz 15. z poprzedniego rozdziału dotyczą programistów algorytmów zarówno w trybie zwykłym jak i krokowym.
  3. W przypadku algorytmów w trybie krokowym nie należy nadpisywać implementacji metody findAndShowResult(), która tworzy okno „Krokowe wykonywanie algorytmu” pokazane na rysunku ??. Dla algorytmów w trybie krokowym dedykowana jest metoda runOneStep(), która wykonuje 1 krok algorytmu. Poniżej znajduję się przykład implementacji tej metody dla algorytmu Dijksty.
    1@Override 
    2protected void runOneStep() { 
    3    if (first) 
    4        firstStep(); 
    5    else { 
    6        if (!endVertex.equals(vertexU)) { 
    7            if (edgesList.isEmpty() && edgeVchecked) 
    8                selectNextVertex(); 
    9            else { 
    10                if (edgeVchecked) 
    11                    selectNextEdge(); 
    12                else 
    13                    checkDistance(); 
    14            } 
    15        } else { 
    16            createTextResult(); 
    17            changeState(State.End); 
    18        } 
    19    } 
    20}

    Fragment pochodzi z klasy DijkstraAlgorithm z pakietu modgraf. algorithm.steps. Pełny kod tej klasy można znaleźć w repozytorium Modgrafa na GitHubie.

  4. Największą trudnością w pisaniu algorytmów w trybie krokowym jest brak możliwości używania pętli, które są nieodłączną cechą wszystkich algorytmów. Często w ramach jednej iteracji wykonywanych jest wiele operacji. W algorytmie w trybie krokowym każda taka czynność powinna odpowiadać jednemu stanowi implementowanej klasy. Przed rozpoczęciem implementacji warto zaplanować maszynę stanową dla wybranego algorytmu. W powyższym przykładzie widać „drabinkę ifów”, która jest nielubiana przez doświadczonych programistów, ale bez niej trudno byłoby mi sprawdzić w jakiem stanie znajduje się aktualnie algorytm. Oczywiście możliwe są inne sposoby implementacji maszyny stanowej. Ważne jest jednak, aby każdemu stanowi odpowiadało wywołanie jakiejś metody. Ułatwia to testowanie działania algorytmu.
  5. W każdym kroku powinno znaleźć się modyfikowanie warstwy matematycznej grafu, stanu algorytmu oraz warstwy wizualnej grafu. Przykład:
    1private void checkDistance() { 
    2    // Linie od 3. do 8. modyfikują warstwę matematyczną 
    3    double distanceThroughU = distances.get(vertexU.getId()) 
    4            + ((WeightedEdge)edgeV).getWeight(); 
    5    if (distanceThroughU < distances.get(vertexV.getId())) { 
    6        queue.remove(new DistanceVertex(vertexV, 0.0, null)); 
    7        distances.put(vertexV.getId(), distanceThroughU); 
    8        queue.add(new DistanceVertex(vertexV, distanceThroughU, edgeV)); 
    9        showDistances(); //ta metoda pokazuje w warstwie wizualnej zamiany 
    10                         //wykonane w warstwie matematycznej 
    11    } 
    12    edgeVchecked = true; //ta flaga zmienia stan algorytmu 
    13}
